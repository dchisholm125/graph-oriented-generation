# Public / Private Boundary

This repository is the public GOG Lite repository. It contains the reference
implementation, public benchmark harnesses, public thesis, and conservative
summaries of selected private benchmark results.

The private GOG Professional repository contains the research/product engine and
implementation evidence that should not be copied into this repo.

## Public: GOG Lite

This public repository may contain:

- GOG Lite reference code in `gog_engine_lite/`.
- Public benchmark helpers in `gog/` and `gog_cli/`.
- Public result artifacts generated by the public harness.
- Design notes explaining the graph/cartography thesis.
- Conservative result summaries from GOG Professional benchmarks.
- Links to the public project page and public documentation.

GOG Lite is deliberately simple: regex import graphs, keyword seeding, bounded
graph expansion, transparent caps, and public scoring rules. It exists for
transparency, research, and reproducibility.

## Private: GOG Professional

The private repository contains:

- Production context strategy and repo cartography.
- Benchmark harnesses and private case-study artifacts.
- Proprietary context strategy, semantic-hazard synthesis, and bad-path
  generation.
- Surgical mutation contracts, deterministic operation application, and
  mutation-safety experiments.
- Assistant integration experiments, including OpenCode bakeoffs.
- Raw transcripts, prompts, diagnostics, local artifact layouts, and internal
  implementation evidence.

These materials may inform public summaries, but they should not be copied into
this repository.

## What Public Docs May Summarize

Public docs may say, at a high level, that:

- GOG Professional matched validated outcomes from hybrid RAG with smaller
  context in controlled benchmarks.
- GOG semantic-hazard guidance helped OpenCode avoid a previously observed
  TypeScript typecheck failure.
- Hybrid RAG can also pass; GOG's observed advantage so far is context
  efficiency, repo locality, bad-path guidance, and reduced repo archaeology.
- GOG Lite is public while GOG Professional remains private.

Public docs should avoid raw private artifacts and sensitive internals.

## What Must Not Be Published Here

Do not copy:

- Private source code.
- Private implementation details.
- Raw private benchmark artifacts.
- Local absolute paths.
- Raw prompts or transcripts from private runs.
- Internal benchmark harness code.
- Proprietary ranking, pruning, context-strategy, or semantic-hazard generation
  logic.
- Sensitive repository internals from private test fixtures.

When in doubt, summarize behavior and outcome, not mechanism.

## Boundary Check Before Public Changes

Before publishing docs or code in this repository, verify:

- The repo still runs without private imports.
- No private source files were copied.
- No local machine paths or artifact roots are included.
- Benchmark numbers are framed as controlled, task-local evidence.
- RAG is not presented as incapable or obsolete.
- GOG is not presented as a coding assistant replacement.
