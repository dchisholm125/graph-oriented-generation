# Contributing to Graph-Oriented Generation (GOG)

GOG is an active research prototype for repo-resident symbolic state, explicit reasoning interfaces, and bounded language rendering.

### How Can I Contribute?
1. **Repository onboarding:** Help turn one-off graph seeding into a durable, refreshable onboarding workflow.
2. **Language parsers:** Extend symbolic extraction beyond the current TypeScript/Vue focus.
3. **Reasoner contracts:** Improve typed mutation plans, refusal states, and validation requirements.
4. **Benchmarks:** Add public-repo tasks with executable validation, not only string rubrics.
5. **Renderer studies:** Test how small models behave when architecture is made explicit upstream.

### Pull Request Process
- Preserve the distinction between repo state, reasoning, rendering, and validation.
- Add focused tests or validation evidence when changing planner, parser, or benchmark behavior.
- Explain graph or benchmark changes concretely enough that another researcher can reproduce the reasoning.
